## Why Multiple Platforms?

We believe developers should access documentation in the way that best fits their workflow:

### 🌐 **Web Browser**
Perfect for detailed reading, visual learning, and exploring documentation while coding.

### 📱 **Mobile App**
Ideal for learning on the go, commuting, or reviewing documentation away from your desk.

### ⚡ **Command Line**
Best for quick lookups, automation, scripting, and integrating documentation into your development workflow.

### 🤖 **AI Assistants**
Let AI help you navigate and understand documentation through natural language queries.

## Cross-Platform Features

### Unified Authentication

Single sign-on works across all platforms:

```typescript
// Same authentication flow everywhere
const auth = useAuth();
await auth.signIn({ provider: 'google' });
```

### Data Synchronization

Your bookmarks, history, and preferences sync automatically:

- Bookmark a page on mobile → Available on CLI and web
- Update preferences on web → Synced to mobile app
- Search history → Shared across all platforms

### Consistent APIs

Same API structure across all platforms:

```bash
# CLI
zudoku search "authentication"

# MCP
{"tool": "search_documentation", "query": "authentication"}

# Mobile
docRepository.search("authentication")
```
