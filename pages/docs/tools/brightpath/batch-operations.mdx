# Batch Operations

Batch operations allow you to execute multiple AI agent tasks in parallel or sequence, monitor their progress, and merge the results automatically. This is ideal for large-scale automation workflows like updating documentation across multiple repositories, triaging issues, or performing code reviews.

## Overview

A batch operation consists of:

1. **Batch File**: A text file defining all tasks to execute
2. **Launch**: Starting the batch operation
3. **Monitor**: Tracking progress of individual tasks
4. **Merge**: Combining results from completed tasks

## Batch File Format

Batch files use a simple text format where each task is separated by a blank line. Each task can specify:

- `repo`: Repository identifier
- `task`: Task description or prompt
- `budget`: Budget allocation for this task
- `provider`: Provider to use (optional, uses default if not specified)
- `model`: Model to use (optional)
- `timeout`: Timeout in seconds (optional)

### Basic Format

```text
repo: project-name
task: Task description here
budget: 2.00

repo: another-project
task: Another task description
budget: 1.50
```

### Full Format with All Options

```text
repo: zudoku-docs
task: Update API reference documentation with latest changes
budget: 3.00
provider: cursor
model: gpt-4
timeout: 300

repo: brightpath-cli
task: Add installation instructions for Windows platform
budget: 2.50
provider: cursor
model: gpt-4
timeout: 200
```

## Launching Batch Operations

### Basic Launch

```bash
# Launch a batch with default settings
brightpath batch launch docs-update-batch.txt
```

### Launch with Options

```bash
# Launch with custom settings
brightpath batch launch docs-update-batch.txt \
  --provider cursor \
  --max-concurrent 5 \
  --budget 50.00 \
  --name "Q1 Documentation Update" \
  --output-dir ./batch-results
```

### Launch Options

| Flag | Description | Default |
|------|-------------|---------|
| `--provider` | Default provider for all tasks | From config |
| `--max-concurrent` | Maximum concurrent tasks | 5 |
| `--budget` | Total budget limit | Sum of task budgets |
| `--name` | Name for this batch operation | Auto-generated |
| `--output-dir` | Directory for output files | `./batch-output` |
| `--dry-run` | Validate without executing | `false` |
| `--notify` | Notification webhook URL | None |

### Dry Run

Before launching a batch, validate it with a dry run:

```bash
# Validate batch file and show what would be executed
brightpath batch launch docs-update-batch.txt --dry-run
```

Output:


## Merging Results

After tasks complete, merge the results into a cohesive output.

### Automatic Merge

```bash
# Merge with automatic strategy
brightpath batch merge batch-abc123 --strategy auto
```

### Merge Strategies

#### Auto Strategy

Automatically resolves conflicts based on:
- Task completion time (latest wins)
- Task success rate
- Confidence scores from AI agents

```bash
brightpath batch merge batch-abc123 --strategy auto
```

#### Manual Strategy

Prompts for manual resolution of conflicts:

```bash
brightpath batch merge batch-abc123 --strategy manual
```

#### Consensus Strategy

Uses consensus from multiple task results:

```bash
brightpath batch merge batch-abc123 --strategy consensus --threshold 0.7
```

#### Latest Strategy

Uses the most recent successful result:

```bash
brightpath batch merge batch-abc123 --strategy latest
```

### Merge Options

```bash
# Merge with custom options
brightpath batch merge batch-abc123 \
  --strategy consensus \
  --conflicts prompt \
  --output ./merged-results \
  --create-pr \
  --pr-title "Documentation Updates Q1"
```

### Handling Merge Conflicts

When conflicts occur, Brightpath can:

1. **Skip**: Skip conflicting changes
2. **Prompt**: Ask for manual resolution
3. **Override**: Use a specific strategy to resolve

```bash
# Handle conflicts with prompts
brightpath batch merge batch-abc123 --conflicts prompt

# Skip conflicts
brightpath batch merge batch-abc123 --conflicts skip

# Override with latest
brightpath batch merge batch-abc123 --conflicts override --override-strategy latest
```

## Advanced Batch Operations

### Conditional Execution

Use conditions to control task execution:

```text
repo: backend-api
task: Update API documentation
budget: 2.00
condition: tests-passing

repo: backend-api
task: Deploy to staging
budget: 3.00
condition: docs-updated
depends_on: task-1
```

### Task Dependencies

Define dependencies between tasks:

```text
repo: frontend-app
task: Update dependencies
budget: 1.00
id: update-deps

repo: frontend-app
task: Run tests
budget: 2.00
depends_on: update-deps

repo: frontend-app
task: Build production
budget: 1.50
depends_on: update-deps,run-tests
```

### Retry Configuration

Configure retry behavior for failed tasks:

```text
repo: api-service
task: Deploy to production
budget: 5.00
retry_attempts: 3
retry_delay: 60
retry_backoff: exponential
```

## Best Practices

### Budgeting

- **Set per-task budgets**: Prevent runaway costs
- **Use total budget limit**: Cap overall spending
- **Monitor in real-time**: Watch costs as batch executes

```bash
brightpath batch launch batch.txt \
  --budget 100.00 \
  --warn-threshold 80.00 \
  --stop-on-limit
```

### Task Granularity

- **Break large tasks into smaller ones**: Easier to monitor and debug
- **Use task dependencies**: Control execution order
- **Group related tasks**: Organize by repository or feature

### Error Handling

- **Enable auto-retry**: Recover from transient failures
- **Set timeouts**: Prevent hanging tasks
- **Use notifications**: Get alerts on failures

```bash
brightpath batch launch batch.txt \
  --retry-failed \
  --timeout 300 \
  --notify https://hooks.slack.com/services/YOUR/WEBHOOK/URL
```

### Output Management

- **Use output directories**: Organize results by batch
- **Enable logging**: Debug issues more easily
- **Save intermediate results**: Checkpoint progress

```bash
brightpath batch launch batch.txt \
  --output-dir ./results/q1-docs \
  --log-level debug \
  --checkpoint-interval 5
```

## Troubleshooting

### Batch Won't Start

```bash
# Validate batch file
brightpath batch launch batch.txt --dry-run

# Check configuration
brightpath config validate

# Test provider connectivity
brightpath provider test cursor
```

### Tasks Failing

```bash
# View detailed task logs
brightpath batch status batch-abc123 --task 3 --logs

# Check task configuration
brightpath batch inspect batch-abc123 --task 3

# Retry failed tasks
brightpath batch retry batch-abc123 --failed-only
```

### Merge Issues

```bash
# Preview merge without applying
brightpath batch merge batch-abc123 --dry-run

# Use different merge strategy
brightpath batch merge batch-abc123 --strategy manual

# Export results for manual merge
brightpath batch export batch-abc123 --format json
```
