## 1. CrewAI Flows

Flows are the event-driven orchestration layer for chaining Crews, tasks, and arbitrary Python
logic.

### Imports

```python
from crewai.flow.flow import Flow, start, listen, router, or_, and_
from crewai.flow.human_feedback import human_feedback, HumanFeedbackResult
```

### `@start()` Decorator

Marks entry points. Multiple `@start()` methods run in parallel.

```python
@start()                          # Unconditional start
@start("method_or_label")        # Gate on prior method or router label
@start(lambda self: condition)    # Conditional callable
```

### `@listen()` Decorator

Triggers when a specified method completes. Receives that method's return value as argument.

```python
@listen(generate_city)            # Listen to method reference
@listen("generate_city")          # Listen to method name string
@listen("approved")               # Listen to router label string
```

### `@router()` Decorator

Defines conditional routing. Returns a string label that `@listen` handlers match.

```python
@router(start_method)
def route_decision(self):
    if self.state.success_flag:
        return "success"
    else:
        return "failed"

@listen("success")
def on_success(self): ...

@listen("failed")
def on_failure(self): ...
```

### `or_()` and `and_()`

Control flow combinators for listening to multiple methods.

```python
from crewai.flow.flow import or_, and_

@listen(or_(method_a, method_b))   # Fires when ANY of the methods complete
def handler(self, result): ...

@listen(and_(method_a, method_b))  # Fires when ALL of the methods complete
def handler(self): ...
```

### `@human_feedback` Decorator

**Requires CrewAI >= 1.8.0**. Pauses execution for human review.

```python
from crewai.flow.human_feedback import human_feedback, HumanFeedbackResult

@start()
@human_feedback(
    message="Do you approve this content?",       # Prompt shown to user
    emit=["approved", "rejected", "needs_revision"],  # Routing outcomes
    llm="gpt-4o-mini",                           # LLM to interpret feedback
    default_outcome="needs_revision",             # Fallback if no feedback
)
def generate_content(self):
    return "Content to review..."

@listen("approved")
def on_approval(self, result: HumanFeedbackResult):
    print(f"Feedback: {result.feedback}")
    print(f"Output: {result.output}")
```

Access feedback history: Access feedback history:

- `self.last_human_feedback` — most recent
- `self.human_feedback_history` — all feedback as a list

## 2. CrewAI Memory

Unified `Memory` class replacing separate short-term, long-term, entity, and external memory.

### Constructor

```python
from crewai import Memory

memory = Memory(
    llm="gpt-4o-mini",                    # Default. LLM for analysis
    storage="lancedb",                     # "lancedb", path string, or StorageBackend instance
    embedder=None,                         # Config dict, callable, or None (OpenAI default)
    recency_weight=0.3,                    # Weight for recency in composite score
    semantic_weight=0.5,                   # Weight for semantic similarity
    importance_weight=0.2,                 # Weight for importance
    recency_half_life_days=30,             # Days for recency to halve
    consolidation_threshold=0.85,          # Similarity above which consolidation triggers
    consolidation_limit=5,                 # Max records to compare during consolidation
    default_importance=0.5,               # Importance when LLM analysis skipped
    batch_dedup_threshold=0.98,           # Cosine similarity for intra-batch dedup
    confidence_threshold_high=0.8,         # Recall confidence for direct return
    confidence_threshold_low=0.5,         # Recall confidence for deeper exploration
    complex_query_threshold=0.7,          # Complex query exploration threshold
    exploration_budget=1,                 # LLM-driven exploration rounds in deep recall
    query_analysis_threshold=200,         # Chars below which LLM analysis is skipped
)
```

### Core Methods

```python
# Store — LLM infers scope, categories, importance
memory.remember("We chose PostgreSQL for the user database.")
memory.remember("Sprint velocity is 42 points", scope="/team/metrics")
memory.remember(content, source="user:alice", private=True)

# Batch store (non-blocking, background thread)
memory.remember_many(["Fact A.", "Fact B.", "Fact C."])

# Retrieve — ranked by composite score
matches = memory.recall("What database did we choose?")
matches = memory.recall(query, limit=10, depth="shallow")  # No LLM
matches = memory.recall(query, limit=10, depth="deep")     # Default, LLM-guided
matches = memory.recall(query, scope="/project/beta")
matches = memory.recall(query, source="user:alice")

# Extract atomic facts from text
facts = memory.extract_memories("Meeting notes: We decided to migrate...")

# Forget
memory.forget(scope="/project/old")

# Reset
memory.reset()                    # All scopes
memory.reset(scope="/project/old")  # Subtree only
```

### Scope Operations

```python
# Create scoped view (restricts all ops to subtree)
agent_memory = memory.scope("/agent/researcher")
agent_memory.remember("Found three papers.")   # Stored under /agent/researcher
agent_memory.recall("papers")                  # Searches only /agent/researcher

# Subscope
project_memory = agent_memory.subscope("project-alpha")  # /agent/researcher/project-alpha
```

### Memory Slices

Cross-scope views (multiple disjoint branches):

```python
# Read-only slice
view = memory.slice(
    scopes=["/agent/researcher", "/company/knowledge"],
    read_only=True,
)
matches = view.recall("security policies", limit=5)
# view.remember("...") → raises PermissionError

# Read-write slice
view = memory.slice(scopes=["/team/alpha", "/team/beta"], read_only=False)
view.remember("Cross-team decision", scope="/team/alpha")
```

### Discovery

```python
memory.tree()                           # Full scope tree with record counts
memory.tree("/project", max_depth=2)    # Subtree
memory.info("/project")                 # ScopeInfo: record_count, categories, dates
memory.list_scopes("/")                 # Immediate child scopes
memory.list_categories()                # Category names and counts
memory.list_records(scope="/project/alpha", limit=20)
```


```python
content_source = CrewDoclingSource(

```

```python
csv_source = CSVKnowledgeSource(file_paths=["data.csv"])
```

### TextFileKnowledgeSource / PDFKnowledgeSource


```python
json_source = JSONKnowledgeSource(file_paths=["data.json"])
```

```python

```

### Custom Knowledge Source

Extend `BaseKnowledgeSource`:

```python
from crewai.knowledge.source.base_knowledge_source import BaseKnowledgeSource

class CustomSource(BaseKnowledgeSource):
    def load_content(self) -> str:
        # Return your custom content
        return "Custom knowledge content..."

    def add(self) -> None:
        content = self.load_content()
        chunks = self._chunk_text(content)
        self.chunks.extend(chunks)
        self.save_documents(metadata=self.metadata)
```

### CLI

```bash
crewai train -n <n_iterations> -f <filename>
```

    {
      flag: "`-f, --filename`",

    },
  ]}

### Programmatic

```python
crew = YourCrew().crew()
crew.train(
    n_iterations=2,
    inputs={"topic": "AI"},
    filename="your_model.pkl"
)
```

### How It Works

For each iteration, per agent:

1. Agent produces `initial_output`
2. You provide `human_feedback` inline
3. Agent produces `improved_output` based on feedback
4. Results saved to `training_data.pkl` (working) and final `trained_agents_data.pkl`

After training, suggestions are automatically loaded and appended to task prompts during normal
`kickoff()`.

```
│   │   └── tasks.yaml           # Task definitions

```

### agents.yaml

```yaml
researcher:
  role: >
    {topic} Senior Data Researcher
  goal: >
    Uncover cutting-edge developments in {topic}
  backstory: >
    You're a seasoned researcher with a knack for uncovering developments in {topic}.
  llm: openai/gpt-4o # Optional: override default LLM
  verbose: true
```

### tasks.yaml

```yaml
research_task:
  description: >
    Conduct thorough research about {topic}
  expected_output: >
    A list with 10 bullet points of relevant information about {topic}
  agent: researcher
  markdown: true
  output_file: report.md
```

Variables like `{topic}` are replaced via `crew.kickoff(inputs={"topic": "AI"})`.

### Decorators

```python
from crewai import Agent, Crew, Process, Task
from crewai.project import CrewBase, agent, task, crew, before_kickoff, after_kickoff

@CrewBase
class MyCrew:
    agents_config = "config/agents.yaml"
    tasks_config = "config/tasks.yaml"

    @before_kickoff
    def prepare_inputs(self, inputs):
        inputs["extra"] = "data"
        return inputs

    @after_kickoff
    def process_output(self, output):
        output.raw += "\nPost-processed."
        return output

    @agent
    def researcher(self) -> Agent:
        return Agent(
            config=self.agents_config["researcher"],
            tools=[SerperDevTool()],
            verbose=True
        )

    @task
    def research_task(self) -> Task:
        return Task(config=self.tasks_config["research_task"])

    @crew
    def crew(self) -> Crew:
        return Crew(
            agents=self.agents,    # Auto-collected by @agent
            tasks=self.tasks,      # Auto-collected by @task
            process=Process.sequential,
            verbose=True
        )
```

## 8. CrewAI Pipelines / Multi-Crew Orchestration

As of 2025-2026, the primary multi-crew orchestration mechanism is **Flows** (see Section 1). The
older `Pipeline` class has been superseded by Flows.

### Flows as Pipelines

```python
from crewai.flow.flow import Flow, start, listen

class MultiCrewPipeline(Flow):
    @start()
    def research_phase(self):
        return ResearchCrew().crew().kickoff(inputs=self.state)

    @listen(research_phase)
    def analysis_phase(self, research_output):
        return AnalysisCrew().crew().kickoff(inputs={"research": research_output.raw})

    @listen(analysis_phase)
    def writing_phase(self, analysis_output):
        return WritingCrew().crew().kickoff(inputs={"analysis": analysis_output.raw})
```

Scaffolding: `crewai create flow name_of_flow` generates the folder structure with `crews/`
directory.

## 10. CrewAI Telemetry & Callbacks

### Crew Callbacks

```python
def my_step_callback(step_output):
    print(f"Agent step: {step_output}")

def my_task_callback(task_output):
    print(f"Task completed: {task_output.description}")

crew = Crew(
    agents=[...],
    tasks=[...],
    step_callback=my_step_callback,     # Called after each agent step
    task_callback=my_task_callback,     # Called after each task completion
    output_log_file=True,              # logs.txt in cwd
    # output_log_file="custom.json",   # JSON format if .json extension
    # output_log_file="custom.txt",    # Text format otherwise
    verbose=True,
    share_crew=False,                  # Share data with CrewAI team
)
```

### Agent Step Callback

```python
agent = Agent(
    role="...", goal="...", backstory="...",
    step_callback=my_agent_callback,    # Overrides crew's step_callback for this agent
)
```

### Task Callback

```python
task = Task(
    description="...", expected_output="...", agent=agent,
    callback=my_task_callback,          # Per-task callback
)
```

### Telemetry

Anonymous telemetry enabled by default. Disable with:

```bash
CREWAI_DISABLE_TELEMETRY=true
# or
OTEL_SDK_DISABLED=true
```

Data collected: CrewAI/Python versions, crew metadata, agent roles, task metadata, tool usage,
execution data.

### Execution Hooks / Events

```python
from crewai.events import BaseEventListener, MemoryQueryCompletedEvent

class MyMonitor(BaseEventListener):
    def setup_listeners(self, crewai_event_bus):
        @crewai_event_bus.on(MemoryQueryCompletedEvent)
```

## 12. CrewAI Guardrails

### Task Parameters

```python
task = Task(
    description="Write a blog post",
    expected_output="A blog post under 200 words",
    agent=blog_agent,
    guardrail=validate_function,          # Single guardrail
    guardrails=[fn1, fn2, "LLM string"],  # Multiple (overrides guardrail)
    guardrail_max_retries=3,              # Default: 3. Max retries on failure
)
```

### Function-Based Guardrail Signature

```python
from crewai import TaskOutput
from typing import Tuple, Any

def my_guardrail(result: TaskOutput) -> Tuple[bool, Any]:
    """
    Args:
        result: TaskOutput with .raw, .pydantic, .json_dict
    Returns:
        (True, validated_result) on success
        (False, "error message") on failure
    """
    if len(result.raw.split()) > 200:
        return (False, "Content exceeds 200 words")
    return (True, result.raw.strip())
```

### LLM-Based Guardrail (String)

```python
task = Task(
    description="Write a blog post",
    expected_output="...",
    agent=blog_agent,
    guardrail="The blog post must be under 200 words and contain no jargon"
)
```

### Multiple Guardrails (Mixed)

```python
task = Task(
    ...,
    guardrails=[
        validate_word_count,                              # Function
        "Content must be engaging for general audience",   # LLM string
        "Writing style should be clear and concise",       # LLM string
    ],
    guardrail_max_retries=3
)
```

Guardrails execute sequentially. If any fails, error goes back to agent for retry.

**Note**: `max_retries` is deprecated → use `guardrail_max_retries`.
