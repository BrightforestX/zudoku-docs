# Errors — brightpath_swarm

Error handling strategy for the brightpath_swarm controller and documentation workflows.

## Error handling strategy

- **Fail fast** — Validation and path checks before sync/scripts run.
- **Logging** — Scripts log failures with context (repo name, path, error message).
- **User-facing messages** — Clear output when sync fails (e.g. missing repo, permission denied).

## Error catalog

| Condition            | Handling                                | Runbook                              |
| -------------------- | --------------------------------------- | ------------------------------------ |
| Repo path not found  | Exit with message; skip repo            | Verify workspace-repos.json paths    |
| Sync script fails    | Log and exit; Docusaurus build may fail | Check sync-docs.js skip rules        |
| MDX parse error      | File skipped; build continues           | See TROUBLESHOOTING.md               |
| Wireframe copy fails | Log and continue                        | Verify PathX_Matcher wireframes path |
